EzriRx - EDI 812 Credit/Debit Adjustment Specifications

The below documentation outlines the requirements for an 812 EDI file that represents a Credit/Debit Adjustment.

As a wholesaler, you can supply an 812 to represent a Credit/Debit Adjustment. It can be used to communicate when a deduction (short pay) has occurred or a credit or debit is expected. It identifies and contains the details and amounts covering exceptions, adjustments, credits, or debits.

What are 812 Credit/Debit Adjustment used for?

  1. Items were defective, damaged or spoiled
  2. Items will be or have been returned
  3. The wrong products were sent
  4. Products were not received
  5. The number of items delivered was more or less than the amount ordered
  6. An adjustment or deduction has occurred for rebates or allowances
  7. There was a pricing error on the original PO or invoice

1. Returns + Price Adjustments

Here is an example file that satisfies EzriRx's requirements for an 812 EDI file which represents Returns + Price Adjustments: ezrirx_812_returns_price_adjustments_example.edi

Read below for a breakdown of each segment and other details. All samples used below are based off the example file provided above.

File Requirements

Note: if you provide us with an 832 or 856, you are likely following these requirements already:

  • Please provide a .edi file with a unique name for each import - a timestamp is recommended (i.e. "812_1518538502.edi").
  • In your FTP server please create an outbound directory in the root directory where you will drop EDI files. It can be named anything you choose.
  • You can drop files into the outbound FTP directory as frequently as you'd like. We scan for new files on an hourly basis.

Here are some important details about the information that needs to be provided in these segments:

  • ISA: make sure the ISA states your vendor name and "EZRIRX" as shown in the example.
  • ST: ensure that the value of ST01 is 812. ST02 can have any value.
  • BCD:
    • BCD01 Credit/Debit adjustment date. (YYYYMMDD)
    • BCD03 Transaction Handling Code. Code designating the action to be taken. Must be O for Returns + Price Adjustments
      • D Make Payment Only
      • E Debit/Credit Advice with Remittance Detail
      • J Payment Requested (Adjustment to Account)
      • K Reimbursement to Follow
      • L Apply on Account
      • O Deduct from Next Remittance
      • P Prenotification of Future Transfers
      • Q Credit Due, Payment Being Issued
      • S Single Debit Authorization
      • U Split Payment and Remittance
    • BCD04 Net amount of this complete transaction.
    • BCD05 Credit/Debit Flag Code. Code indicating whether amount is a credit or debit. Must be C for Returns + Price Adjustments.
      • C for Credit transaction
      • D for Debit transaction
    • BCD08 Ezrirx Order Number. (Example Order Number: R123456789)

  • CDD01: Adjustment Reason Code. Must be 72 for Returns + Price Adjustments.
    • 72 Authorized Return
    • 79 Cooperative Advertising 81 Credit as Agreed
    • 81 Credit as Agreed
    • 84 Deviation from Order Date (early shipment)
    • 86 Duplicate Payment
    • 87 Duplicate Shipment
    • 99 Late Shipment of Goods
    • A2 Misshipped (Wrong Product Received)
    • A5 Overage
    • B4 Refurbishing Charge
    • B7 Restocking Charge
    • D3 Unauthorized Deduction
    • D7 Cost Associated with Reworked Material E9 Order Cancelled
    • E9 Order Cancelled
  • CDD02 Must be C for Returns + Price Adjustments.
    • C Credit
    • D Debit
  • CDD04: Total Credit Amount.
  • CDD07: Credit/Debit Quantity.
  • CDD11: Unit Price.
  • LIN02: Use this segment to specify NDC (ND).
  • LIN04: Use this segment to specify SKU (VP) - Vendor Item Number.
  • N901: Reference Identification Qualifier. (RZ)
